## Программа: Конвертер секунд в формат ЧЧ:ММ:СС

### Описание:
Программа принимает от пользователя целое число (количество секунд) и преобразует его в стандартный временной формат **часы:минуты:секунды** (HH:MM:SS). Результат выводится с ведущими нулями для сохранения двузначного формата.

### Принцип работы:
Программа использует операции целочисленного деления (`/`) и остатка от деления (`%`) для выделения часов, минут и секунд из общего количества секунд:

- **Часы** = (общее количество секунд / 3600) % 24  
  (деление на 3600 даёт количество часов, операция `% 24` ограничивает значение до 24-часового формата)

- **Минуты** = (общее количество секунд % 3600) / 60  
  (сначала находим остаток секунд после вычитания часов, затем делим на 60)

- **Секунды** = общее количество секунд % 60  
  (остаток от деления на 60 даёт последние секунды)

### Пример работы программы:

**Ввод:** `3661` (1 час, 1 минута, 1 секунда)  
**Вывод:** `01:01:01`

**Ввод:** `125` (2 минуты, 5 секунд)  
**Вывод:** `00:02:05`

### Особенности реализации:
- Формат вывода `%02d` обеспечивает отображение чисел с ведущим нулём (например, `05` вместо `5`)
- Программа поддерживает значения больше 24 часов (например, 90000 секунд = 25 часов выведется как `01:00:00` из-за `% 24`)

### Используемые технологии:
- **Язык программирования:** C (стандарт C99/C11)
- **Библиотека:** `stdio.h` (для ввода/вывода)
- **Операции:** `/` (целочисленное деление), `%` (остаток от деления)
- **Функции:** `scanf()` (ввод данных), `printf()` (вывод результата)
- **Форматирование:** `%02d` (двузначное число с ведущим нулём)
- **Компилятор:** GCC / MinGW / Clang / любой другой компилятор Си

### Объяснение результата:

#### Почему адреса отличаются на 4 байта?
- Тип `int` в большинстве современных систем занимает **4 байта** (32 бита)
- Компилятор размещает переменные в стеке последовательно
- Обычно переменные располагаются **сверху вниз** (с уменьшением адресов) или наоборот

#### Почему именно такой порядок?
В данном примере адреса уменьшаются:
- `c` имеет самый маленький адрес (0x...24)
- `b` больше на 4 (0x...28)
- `a` больше на 4 (0x...2c)

Это означает, что компилятор разместил переменные в стеке в порядке их объявления, но с выделением памяти "вниз" (от старших адресов к младшим).

   1. Как объявить структуру в языке C?

Структура объявляется с помощью ключевого слова struct, после которого идёт имя структуры, а в фигурных скобках перечисляются поля с их типами. В конце обязательно ставится точка с запятой.

2. Что такое вложенные структуры и зачем они нужны?

Вложенные структуры — это когда внутри одной структуры в качестве поля используется другая структура. Они нужны для логической группировки данных, удобства работы и повторного использования кода. Например, в структуре «Студент» можно вложить структуру «Дата рождения».

3. Как передать структуру в функцию?

Структуру можно передать двумя способами:

По значению — в функцию передаётся копия структуры. Оригинал при этом не меняется.

По указателю — в функцию передаётся адрес структуры. Это экономит память и позволяет изменять оригинал внутри функции.

4. Можно ли создавать массивы структур?

Да, можно. Создаётся обычный массив, где каждый элемент является структурой. Например, массив из 30 студентов.

5. Как обратиться к элементу массива структур?

Сначала указывается имя массива с индексом в квадратных скобках (выбирается нужный элемент), а затем через точку — имя поля структуры.

6. Можно ли использовать enum внутри структуры?

Да, можно. Поле структуры может иметь тип перечисления enum. Это удобно, когда нужно хранить ограниченный набор значений, например, курс студента (1, 2, 3, 4) или дни недели.

7. Можно ли сравнивать структуры оператором ==?

Нет, нельзя. В C оператор == не работает со структурами. Если нужно сравнить две структуры, приходится сравнивать каждое поле по отдельности.
